$(section `This site has been moved to $(a `https://trinity-engine.codeberg.page` `https://trinity-engine.codeberg.page`)`) $(/section) $(section Introduction)
$(note `This project is a work-in-progress.`)

Trinity is a retro, $(a `2.5D` `https://en.wikipedia.org/wiki/2.5D`) $(a software-rendered `https://en.wikipedia.org/wiki/Software_rendering`) game engine written in $(a C99 `https://en.wikipedia.org/wiki/C99`) using the $(a SDL2 `https://wiki.libsdl.org/SDL2/FrontPage`) library, with $(a Lua `https://www.lua.org/about.html`) for scripting. It's inspired by engines such as $(a `Valve's Source engine` `https://en.wikipedia.org/wiki/Source_(game_engine)`), the $(a `Doom engine` https://en.wikipedia.org/wiki/Doom_engine), the $(a `Build engine` `https://en.wikipedia.org/wiki/Build_(game_engine)`), the $(a `Quake engine` https://en.wikipedia.org/wiki/Quake_engine), and $(a Roblox `https://en.wikipedia.org/wiki/Roblox`).

$(h2 Goal)

The goal is to offer a smooth game development experience with easy game distribution. Trinity will be a single pre-compiled executable - the Trinity engine executable - which can run platform-independent game files.

$(h2 `How The Renderer Works`)

If you're interested in how Trinity's renderer works and the mathematics behind it, you can read the $(a `Trinity renderer development blogs` `https://lordoftrident.github.io/blogs/trinity-part-1/`).

$(/section) $(section `Platform Support`)

Trinity aims to support the following platforms:

  • Windows
  • Linux
  • MacOS
$(note `As of now, Linux is supported and Windows has untested support.`)
$(/section) $(section `Current State`)

Currently, Trinity has the following features:

  • Functional game file browser with proper UI
  • Wall filling
  • Sectors
  • Working portals
  • Unfinished but usable map editor
